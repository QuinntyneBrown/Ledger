Motion & feedback

Motion in Ledger is a messenger, never decoration. The dial sweeps because progress is accruing; the chart draws in because a story is unfolding; a toast rises because something just happened. Every animation earns its place by explaining a change of state — and every one steps aside when the person asks for calm.

The motion tokens

Two scales do all the work: a set of durations from a tap to the hero sweep, and two signature easings. --ease-standard is the neutral workhorse; --ease-emphasized is the springy curve reserved for entrances and reveals.

TokenValueUsed for
--dur-instant80msTaps, checkbox tick — immediate acknowledgement.
--dur-fast140msHover, color & border transitions, button press.
--dur-base220msSwitch thumb, scrim fade, count-up steps.
--dur-slow360msDialog, sheet & toast entrance, staggered rise.
--dur-slower560msChart draw-in.
--dur-dial1100msThe hero gauge sweep — the one indulgent motion.
--ease-standardcubic-bezier(.2,0,0,1)Default for almost everything.
--ease-emphasizedcubic-bezier(.16,1,.3,1)Springy entrances: sheet, rise, dial, thumb.

The reduced-motion contract

Motion is opt-out at the operating-system level, and Aurora honors it without exception. When prefers-reduced-motion: reduce is set, two things happen automatically:

  • Every duration token collapses to 1ms, so transitions resolve instantly instead of animating.
  • The signature entrances — .au-rise, toast, dialog, sheet, scrim and the chart draw-in — have their keyframe animations switched off, and the gauge jumps straight to its final value.

The result: nothing moves, but every final state is identical. No information lives in the animation alone — the dial still shows 73%, the toast still says what happened, the list still appears.

@media (prefers-reduced-motion: reduce) {
  :root {
    --dur-fast: 1ms; --dur-base: 1ms;
    --dur-slow: 1ms; --dur-slower: 1ms; --dur-dial: 1ms;
  }
  .au-rise, .au-toast, .au-dialog, .au-sheet,
  .au-scrim, .au-chart-draw { animation: none !important; }
  .au-gauge .au-gauge-arc { transition: none !important; }
}

Do & don't

Do
Goal reached
Animate a state change — a new entry, a reached goal, a screen arriving.
Don't
Look at me
Don't loop decorative motion — a perpetual wiggle carries no meaning and tires the eye.
Do
Keep it brief. 140–360ms for UI; save the 1.1s sweep for the one hero moment.
Use the shortest duration that still reads — pace to the token scale.
Don't
Don't block the person behind a slow, unskippable transition on every tap.
Never make someone wait on animation to use the app.
Accessibility. Motion is enhancement, not information. Reduced-motion is respected globally, entrances never trap focus or delay interaction, and looping animation is avoided so it can't trigger vestibular discomfort. If a change matters, it's also conveyed in text and final layout — the animation only makes it feel alive.
