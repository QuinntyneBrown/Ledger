Card

A card gathers one subject — a stat, a summary, an action — onto a single elevated surface. It is the building block of the Ledger dashboard: every metric lives in its own quiet frame, so the eye reads one thing at a time.

When to use it

Use a card whenever a piece of content deserves its own boundary — a KPI, a chart, a settings group, a list of recent entries. Cards create rhythm on a busy screen by separating unrelated things without heavy rules or boxes.

  • Use one card per idea. If a card needs two headings, it is probably two cards.
  • Reach for --interactive only when the whole surface navigates somewhere; otherwise keep actions to explicit buttons inside.
  • Use the stat card (.au-stat) for a single number with a trend — it pairs a display numeral with a colored .au-delta.
  • Prefer --glass for cards floating over imagery or the aurora glow; keep plain .au-card for dense grids.

Classes

ClassElementRole
.au-carddiv / aBase surface: --surface-1, hairline border, shadow, top highlight.
.au-card--glassmodifierTranslucent, backdrop-blurred material for floating cards.
.au-card--raisedmodifierDeeper --shadow-3 for a lifted resting state.
.au-card--pad-lg / --pad-smmodifierSwap the default padding for a roomier or denser rhythm.
.au-card--flushmodifierRemoves padding and clips content to the radius (for edge media).
.au-card--interactivemodifierAdds hover lift, brighter border and a pointer cursor.
.au-card-headdivFlex header row: title left, trailing action right.
.au-card-titledivMuted, small, semibold label with an optional leading icon.
