Snackbar

A brief, low-friction confirmation that slides up from the bottom, lingers for a few seconds and dismisses itself. It reassures without blocking — and can carry one action, most often Undo.

Weight logged — 74.9 kg

When to use it

Use a snackbar to confirm that something just happened — a weigh-in saved, a note deleted, a setting changed — when the person doesn’t need to act and shouldn’t be interrupted. It’s the quietest way to close a loop.

  • Keep the message to one short line; the snackbar is a receipt, not a paragraph.
  • Offer at most one action. Undo is the classic — it makes destructive edits feel safe.
  • For a persistent, in-page status use a banner; for a decision that blocks, a dialog.

Anatomy

Message text
1 2 3
  • 1 Icon — tinted by type: success, error or info.
  • 2 Message — a single, plain-language line.
  • 3 Action — optional; a lone brand-colored verb.

Do & don’t

Do
Entry deleted
Pair a destructive result with an immediate Undo so mistakes cost nothing.
Don’t
Sync failed. Retry now or later? Learn more…
Don’t pack a full explanation or a decision into a toast — that belongs in a banner or dialog.

Trigger a snackbar

The data-toast attributes spawn a live toast in the shared region. Press a button — each slides up, waits, then fades on its own.

snackbar/triggers
<button class="au-btn au-btn--primary"
        data-toast="Weight logged — 74.9 kg"
        data-toast-type="success"
        data-toast-action="Undo">Log weight</button>

Types

Three tints map to intent. The message and layout are identical; only the leading icon changes color.

snackbar/types
Weight logged — 74.9 kg
Couldn’t save — you’re offline
Reminder set for 8:00 AM 
<div class="au-toast au-toast--success">
  <svg class="au-icon au-toast-icon"><use href="#i-check-circle"/></svg>
  <span class="au-toast-msg">Weight logged — 74.9 kg</span>
  <button class="au-toast-action">Undo</button>
</div>

From JavaScript

Most toasts are fired in code once an operation resolves. The action’s onAction callback runs before the toast clears.

Aurora.toast("Weight logged — 74.9 kg", {
  type: "success",     // "success" | "error" | "info"
  action: "Undo",       // optional single action
  onAction: restoreEntry,
  duration: 3600        // ms before auto-dismiss
});

Structure & classes

ClassElementRole
.au-toast-regioncontainerFixed, bottom-centered stack. Auto-created on first toast.
.au-toastsurfaceA single snackbar. Add --success, --error or --info.
.au-toast-iconsvgLeading icon, tinted by the type modifier.
.au-toast-msgspanThe message; flexes to fill the row.
.au-toast-actionbuttonOptional lone action in brand color.

Triggers & API

Attribute / callEffect
data-toast="…"On a trigger — the message to show.
data-toast-type="…"Optional type; defaults to success.
data-toast-action="…"Optional action label.
Aurora.toast(msg, opts)Fire from code. opts: type, action, onAction, duration.
Accessibility. The region is an aria-live="polite" status area, so screen readers announce each toast without stealing focus. Keep messages self-contained, never rely on color alone for meaning, and give people enough time — the default life is 3.6 s and the entry animation is disabled under prefers-reduced-motion.
