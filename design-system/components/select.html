Select

Select picks one value from a fixed set. Use the native dropdown for longer lists, and the segmented control when two or three choices deserve to stay visible.

When to use it

Reach for a dropdown select when there are four or more mutually exclusive options, or the list is long enough that inline chips would crowd the layout. Choose the segmented control for two or three peer options that people switch between often — a time range, a unit, a view mode.

  • Order options predictably — alphabetical, numeric, or by frequency.
  • For 2–3 options that toggle a view, prefer the segmented control over a dropdown.
  • Use the brand segment (.is-active.is-brand) to make the active choice unmistakable.
  • If people can pick several values at once, use checkboxes instead.

Anatomy

1 2
  • 1 Field — a 52px control matching the text field’s shell and focus ring.
  • 2 Caret — a non-interactive chevron that signals the native menu.

Do & don’t

Do
Use a segmented control for a small set of peer choices you switch between.
Don’t
Don’t force many options into a segmented control — use a dropdown instead.

Dropdown — default, focus & disabled

The native <select> is styled to match the text field. The middle control simulates the focus ring.

select/states

  <select class="au-select">

    <option>Pounds (lb)</option>
  </select>
  <svg class="au-icon au-select-caret"><use href="#i-chevron-down"/></svg>
</div>

Segmented control

A pill of peer options. The neutral active segment reads as a quiet selection; add .is-brand to make it gold.

select/segmented
<div class="au-segmented">
  <button>Week</button>
  <button class="is-active is-brand">Month</button>
  <button>Year</button>
</div>
