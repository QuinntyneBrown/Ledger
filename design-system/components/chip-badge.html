Chip & badge

Small tokens that label and count. A chip is a compact, often tappable pill for a filter or a status; a badge is a tiny count or dot that rides on an icon; a tag is a mono micro-label for metadata.

All This week On track Plateau Streak 12

When to use each

  • Chip — a filter, a toggleable facet, or a compact status label. Make it --interactive when it filters, and mark the selected one .is-active.
  • Badge — an unread count on an icon button, or a bare --dot to signal “something new” without a number.
  • Tag — an inline, uppercase mono label for metadata such as BETA or a unit; not interactive.
  • Use the semantic variants (--good, --warn, --bad, --brand) to carry meaning, and back them with an icon or word so color isn’t the only cue.

Anatomy

1 2
  • 1 Host — the icon or control the badge annotates.
  • 2 Badge — an absolutely-positioned count pill; use --dot for a bare indicator.

Do & don’t

Do
On track
Pair a semantic color with an icon and a word.
Don’t
Don’t rely on color alone to encode meaning.

Chip classes

ClassRole
.au-chipBase pill: surface-2 background, hairline border, mono-adjacent label.
.au-chip--brandGold-tinted — a highlighted or promoted facet.
.au-chip--good / --warn / --badTeal / amber / coral semantic status.
.au-chip--interactiveAdds hover feedback + pointer for tappable filters.
.au-chip.is-activeSelected state — solid brand fill.

Badge & tag

ClassRole
.au-badgeCount pill (min 20px) — coral fill, white tabular numeral.
.au-badge--dotBare 10px indicator with no number.
.au-tagMono uppercase micro-label on a sunken surface.

Tokens consumed

TokenApplied to
--surface-2 · --borderDefault chip surface + edge.
--brand-subtle · --brandBrand chip and active fill.
--good-subtle / --warn-subtle / --bad-subtleSemantic chip backgrounds.
--bad · --whiteBadge fill and numeral.
--radius-pillChip and badge shape.
Accessibility. Give the badge’s host an aria-label that includes the count (“Notifications, 5 unread”) — the pill is decorative. Interactive chips must be real <button> elements with aria-pressed for the active state, and never encode status with color alone.
