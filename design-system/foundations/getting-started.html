Getting started

Aurora is the design language and component library behind Ledger. One set of tokens, two themes, and a small runtime — everything you need to build calm, precise, data-rich product UI with no dependencies to install.

What Aurora is

Aurora is a self-contained system: pure CSS custom properties, a component stylesheet, and one small JavaScript file that injects icons and wires interactions. There are no build steps, no package installs, and no network calls — it runs from the file system as happily as from a server.

It exists to make one product feel best-in-class: a weight tracker where the numbers are the story. Everything here — the ink canvas, the gold accent, the tabular numerals — is tuned for reading data at a glance, in the dark, without strain.

New here? Skim the five principles below, drop the four asset lines into your <head>, then copy the hello example. You will have a themed, accessible page in under a minute.

Principles

Five ideas shape every decision in Aurora. When a design question comes up, these are the tie-breakers.

One language, two themes

A single token set drives both dark (default) and light. Components never hardcode a colour — flip data-theme and the whole surface follows.

Tabular, data-first

Numbers, dates and deltas render in a tabular mono so digits line up column to column. The "ledger" identity is a typographic decision, not decoration.

Self-contained

No CDNs, no web fonts, no remote images. System-font stacks and an inline icon sprite mean the UI is fast, private and works fully offline.

Calm, then reduced

Motion is purposeful and quiet — a dial that sweeps, a number that counts up. Under prefers-reduced-motion every duration collapses to 1ms.

How to consume it

Add four lines to your document head. Load order matters: tokens first (they define the variables everything else reads), then components, then the page-level stylesheet, then the runtime.

<!-- 1 · design tokens — the single source of truth -->
<link rel="stylesheet" href="assets/tokens.css">
<!-- 2 · the .au-* component library -->
<link rel="stylesheet" href="assets/components.css">
<!-- 3 · the docs chrome (or app.css for the product) -->
<link rel="stylesheet" href="assets/docs.css">
<!-- 4 · the runtime: icons, theme, tabs, gauges, toasts -->
<script defer src="assets/aurora.js"></script>

The page defaults to dark — no attribute required. aurora.js injects the SVG icon sprite on boot, so an icon is just a <use> reference. Keep the four lines together and you inherit theming, focus rings, and every interaction for free.

The token layers

Aurora resolves colour, type and space through three layers. Components reference only the semantic layer, so a single primitive change ripples everywhere — and themes are just a different semantic mapping.

LayerExampleWho reads itRole
1 · Primitives --gold-400 Semantic layer only Raw, theme-agnostic values. The full ink, gold, teal, violet and coral scales. Never referenced by a component directly.
2 · Semantic --brand Components & pages Meaning, not value: --brand, --good, --text-hi, --surface-1. Remapped per theme.
3 · Component --_bg A single component Local, private variables scoped inside a component (the --_ prefix) that let modifiers re-point one part cleanly.
Rule of thumb: never reach past a layer. A component reads --brand, never --gold-400. That single discipline is what makes the light theme a 30-line override instead of a rewrite.
