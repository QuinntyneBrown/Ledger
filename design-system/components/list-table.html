List & table

Two ways to show a sequence of records. The ledger row is a light, scannable line for a feed of entries; the table is for dense, comparable columns. Both set numbers and dates in tabular mono so figures align to the digit.

When to use which

Both render a series of records — the difference is density and intent.

  • Use a list of ledger rows for a chronological feed you scroll: weigh-ins, notes, activity. Each row leads with a mono date, carries a primary value, and can trail a delta.
  • Use a table when the reader compares values across columns — weight vs. body-fat vs. change — or needs a header. Add --zebra for long tables.
  • Make rows interactive only when tapping one opens or edits that record.
  • Every figure uses tabular numerals so the decimal points stack; never mix proportional and tabular numbers in one column.

List classes

ClassElementRole
.au-listdivVertical stack of rows.
.au-rowdivOne record: flex line with a divider below.
.au-row--interactivemodifierHover background + pointer; use when the row navigates.
.au-row-datedivFixed-width mono timestamp, muted.
.au-row-maindivFlexible middle column.
.au-row-primary / .au-row-secondarydivMain value and its subordinate note.
.au-row-valuedivTrailing mono figure; combine with .au-delta.

Table classes

ClassElementRole
.au-tabletableFull-width, collapsed borders, tabular cells.
.au-table--zebramodifierSunken background on even rows for long tables.
th—Uppercase mono label; add inline text-align:right for numeric columns.
td—Tabular numerals by default; dividers between rows.

TokenApplied to

Accessibility. Use a real <table> with <th> headers for comparable data so screen readers announce column context; reserve the list for feeds. Interactive rows should be a <button> or <a> internally, and delta color must be paired with a +/− sign — never color alone.
