Data display

Avatar

A small round mark that stands in for a person or account. It shows a photo when there is one, otherwise initials on an aurora-soft field — always legible, never a broken image.

QB

When to use it

Use an avatar wherever an entry, comment, or profile needs a human face — the account menu, a shared plan, a coach note. In Ledger it is quiet: one in the header, and small ones in shared-goal lists.

  • Prefer a photo when you have one; fall back to initials, then to a generic #i-user icon.
  • Keep initials to one or two letters — the mark must read at 32px.
  • Use the group stack to show several participants compactly; cap it with a “+N” count.
  • Never stretch a non-square image — the component clips to a circle with object-fit:cover.

Sizes

QB QB QB
  • 1 sm — 32px for dense lists and inline mentions.
  • 2 default — 44px for rows and the app header.
  • 3 lg — 72px for a profile header.

Group stack

Overlap avatars with a negative margin and ring each in the canvas color for separation. Cap with a “+N” count.

avatar/group
QB SL MK +4 
<div class="au-row-flex" style="padding-left:12px">
  <span class="au-avatar" style="margin-left:-12px;
        box-shadow:0 0 0 3px var(--surface-1)">QB</span>
  <span class="au-avatar" style="margin-left:-12px;…">SL</span>
  <span class="au-avatar" style="margin-left:-12px;…">+4</span>
</div>
