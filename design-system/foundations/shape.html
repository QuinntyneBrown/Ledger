Shape

Corners carry personality. Aurora rounds generously — soft, friendly, calm — and scales the radius with the surface: tight on small controls, sweeping on sheets. Actions are fully pill-shaped, a signature you can spot at a glance.

The radius scale

Six fixed radii plus pill and circle. Larger elements take a larger radius so the visual "roundness" reads consistently regardless of size.

Shape in practice

The radius is one of the fastest ways to read what a thing is. Pills say "press me"; a 12px corner says "type here"; a big 22–30px corner says "this surface just arrived".

Which corner, where

RadiusBelongs to
--radius-xsTags, code chips, tiny inline swatches, checkbox boxes.
--radius-smNav links, menu items, brand mark, small hover fills.
--radius-mdInputs, selects, banners, code blocks — the workhorse.
--radius-lgCards and panels — the default content container.
--radius-xlDialogs, the FAB, hero surfaces.
--radius-2xlBottom sheets — the big rounded top edge as it slides up.
--radius-pillAll buttons, chips, segmented controls, progress tracks.
--radius-circleAvatars, icon buttons, the gauge, dots and toggles.
Do
Nest a smaller radius inside a larger one so corners stay concentric (md input inside an lg card).
Don't
Don't invert the nesting — a big radius inside a sharp container looks like a mistake, not a choice.
When one rounded element sits inside another, keep them concentric: inner radius ≈ outer radius − padding. Aurora's defaults already line up (md inside lg with space-6 padding).
