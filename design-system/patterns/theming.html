Theming

Every color, shadow and radius in the product resolves through a three-tier token pipeline. Components never see a hex value — they reference intent. Flip one attribute and the entire surface re-themes, because the meaning stays constant while the values underneath swap out.

The token pipeline

Tokens flow one direction. Primitives are raw values with no opinion. Semantic tokens give them a job and are the only layer that changes between themes. Components consume semantic tokens exclusively — so a component written once works in every theme and brand, forever.

theming/pipeline
Tier 1
Primitives
--gold-400: #f9c349 --teal-400: #37e0c8 --ink-800: #12151f
Raw. Never referenced by components.
Tier 2
Semantic
--brand: var(--gold-400) --good: var(--teal-400) --surface-1: var(--ink-800)
The layer that flips per theme.
Tier 3
Component
.au-btn--primary {   background: var(--brand) }
Consumes intent, never hex.
The rule that makes it work. A component may only reference Tier 2. The moment a component hardcodes #f9c349 or reaches for --gold-400 directly, it stops being themeable — and it will look wrong in light mode or under a new brand.

Light & dark

Ledger is dark-first, so the default :root carries the dark semantic values. Light is a pure override block, :root[data-theme="light"], that re-points the same semantic names at lighter values. Nothing else in the codebase knows which theme is live.

:root, :root[data-theme="dark"] {   /* default — dark */
  --surface-1: var(--ink-800);
  --text-hi:   #f4f6fb;
  --brand:     var(--gold-400);
}
:root[data-theme="light"] {          /* override — light */
  --surface-1: #ffffff;
  --text-hi:   #141720;
  --brand:     var(--gold-600);      /* deeper gold for contrast on white */
}

Adding a brand

Because components read intent, reskinning is a Tier-2 job: override the handful of --brand-* tokens (and the aurora stops) and every button, chip, focus ring and gauge follows — without touching a single component rule. Below, the same components run gold and a violet alt-brand side by side.

theming/brand

--brand: violet-500

/* An alternate brand — override intent, not components */
.theme-violet {
  --brand:        var(--violet-500);
  --brand-strong: var(--violet-300);
  --brand-subtle: rgba(124, 92, 255, 0.16);
  --brand-ring:   rgba(124, 92, 255, 0.5);
  --aurora-start: var(--violet-600);   /* re-tune the signature gradient */
}
/* .au-btn--primary, .au-chip--brand, gauges, focus rings — all follow. */

Do & don't

Do
color: var(--text-hi);
background: var(--surface-1);
border: 1px solid var(--border);
Reference semantic tokens — the component adapts to every theme.
Don't
color: #f4f6fb;
background: var(--ink-800);
border: 1px solid #ffffff17;
Don't hardcode hex or reach for Tier-1 primitives — both break light mode.
Accessibility. Each theme sets color-scheme so form controls and scrollbars match, and both palettes are tuned to hold WCAG AA contrast for text and UI. The global toggle persists the choice to localStorage and respects the system preference on first visit.
