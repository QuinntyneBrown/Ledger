Tooltip

A small label that surfaces on hover or keyboard focus to name an icon-only control or add a brief hint. It is CSS-only — no JavaScript, no positioning library — so it costs nothing and works everywhere.

Log a weigh-in

When to use it

A tooltip clarifies — it never carries essential information, because it is invisible on touch and to some assistive tech. Reach for it to name an icon-only button or to explain a terse label.

  • Use it to name icon-only controls and to expand a cramped abbreviation.
  • Keep the text to a few words; a tooltip is a whisper, not a paragraph.
  • Never put an action, link, or the only copy of critical information inside a tooltip.
  • The trigger still needs its own aria-label — the bubble is a visual echo, not the accessible name.

Anatomy

Body-fat % 1 2
  • 1 Bubble — the floating label, positioned above the trigger with a small gap.
  • 2 Trigger — any focusable control wrapped by .au-tooltip.

On an icon button

Wrap any focusable control in .au-tooltip and add a .au-tooltip-bubble sibling.

tooltip/icon-buttons
Edit entry Undo Settings 
<span class="au-tooltip">
  <span class="au-tooltip-bubble">Edit entry</span>
  <button class="au-icon-btn" aria-label="Edit entry">
    <svg class="au-icon"><use href="#i-edit"/></svg>
  </button>
</span>

On a hint control

A tooltip can also explain a label — here an info icon beside a metric. Hover or tab to it.

tooltip/hint
Trend weight A 7-day moving average 
<span class="au-tooltip">
  <span class="au-tooltip-bubble">A 7-day moving average</span>
  <button class="au-icon-btn au-icon-btn--sm"
          aria-label="What is trend weight?">
    <svg class="au-icon"><use href="#i-info"/></svg>
  </button>
</span>

Classes

ClassElementRole
.au-tooltipspanPositioning wrapper around the trigger; reveals the bubble on :hover and :focus-within.
.au-tooltip-bubblespanThe floating label. Positioned above the trigger, hidden until the wrapper is hovered or focused.

Behaviour

TraitDetail
TriggerHover or keyboard focus on any focusable child (:focus-within).
DependenciesNone — pure CSS. aurora.js is not required for tooltips.
PointerThe bubble is pointer-events:none, so it never blocks a click.
Default textSingle line (white-space:nowrap); override for a short wrapped hint.

Accessibility. A tooltip is a visual aid only — it does not appear on touch and may be missed by assistive tech, so the trigger must carry its own aria-label (or visible text). Because it opens on :focus-within, keyboard users get the same hint. Never place the sole copy of essential information, or any interactive element, inside the bubble.
