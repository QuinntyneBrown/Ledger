Iconography

A single stroke set — 24px grid, 2px round-cap strokes, drawn in currentColor. Icons ship as an inline SVG sprite that aurora.js injects on boot, so they render instantly and work fully offline.

How icons work

There are no image requests. Reference a symbol by id and it inherits the current text colour and stroke — theme it, size it and animate it like any other element.

<svg class="au-icon"><use href="#i-target"/></svg>
<!-- aurora.js builds <symbol id="i-target"> … and injects the sprite at boot -->
Offline by design. The sprite is a JavaScript object, not a fetched file — no CDN, no font, no flash of missing glyphs. Load aurora.js and every #i-* reference resolves, even from file://.

Sizes & stroke

Drawn on a 24px grid. Scale to 16 or 20 for dense UI; the 2px stroke stays crisp because it is vector. Below 16px, weight starts to feel heavy — prefer 20/24 wherever space allows.

The filled variant

Icons are outlines by default. Add .au-icon--fill to swap stroke for a solid fill — use it for a selected or active state, like a starred item or an on toggle.

Using icons well

  • Pair an icon with a text label wherever meaning matters — icon-only controls need an aria-label.
  • Icons inherit currentColor: colour them by setting the parent's color, never a hardcoded hex.
  • Keep semantics consistent — i-trending-down is always "good progress" in Ledger, never "loss of data".
  • Optical size > pixel size: a busy glyph like i-settings may need a touch more room than i-check.
Decorative icons should be hidden from assistive tech (the .au-icon class is presentational); meaningful icon-only buttons carry an aria-label. See Accessibility for the full pattern.
